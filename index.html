00

Préparer le terrain

Ouvrir PowerShell, autoriser les scripts, créer son premier fichier et l'exécuter.

🎯 Objectif

Avant d'écrire l'outil, on s'assure que la machine est prête : savoir où on tape les commandes,
comment autoriser un script à s'exécuter et comment lancer un fichier .ps1.
À la fin de cette phase, vous afficherez votre premier message à l'écran.

Qu'est-ce que PowerShell ?

PowerShell est la « console » moderne de Windows : une fenêtre dans laquelle on tape des commandes pour piloter l'ordinateur (lire des informations, lancer des programmes, créer des fichiers…).
Elle est déjà installée sur tous les Windows 10 et 11.
La version livrée d'origine s'appelle Windows PowerShell 5.1 — c'est exactement celle que nous utilisons, sans rien installer.

Ouvrir la console

  1. Appuyez sur la touche ⊞ Windows.
  2. Tapez powershell.
  3. Cliquez sur Windows PowerShell (l'icône bleue).

Une fenêtre s'ouvre, avec une ligne qui se termine par PS C:\Users\VotreNom>.
C'est l'invite de commande : PowerShell attend que vous tapiez quelque chose.

Le tout premier test

Dans la fenêtre, tapez la ligne suivante puis Entrée :

Console PowerShell 
Write-Host "Bonjour MODE 83, je decouvre PowerShell !" -ForegroundColor Yellow

Write-Host veut dire « écris à l'écran ».
Le texte entre guillemets est le message.
-ForegroundColor Yellow demande de l'écrire en jaune.
Vous venez d'exécuter votre première commande !

Ce que vous devez voir dans votre console
Bonjour MODE 83, je decouvre PowerShell !
ℹ️ Pourquoi pas d'accents dans le code ?

Dans une console Windows, les lettres accentuées (é, à, ç…) s'affichent parfois mal selon l'encodage.
Pour éviter tout souci pour les débutants, tous les messages affichés à l'écran sont écrits sans accents
(« decouvre », « systeme »…).
Les accents reviendront proprement plus tard, uniquement dans le fichier Excel, grâce à un encodage spécial (phase 8).

Autoriser l'exécution des scripts

Par sécurité, Windows bloque par défaut le lancement des fichiers de script.
Pour notre formation, on autorise les scripts uniquement pour votre compte utilisateur.
Tapez :

Console PowerShell 
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

Répondez O (Oui) si une question apparaît.
Traduction : « pour mon compte seulement, autorise les scripts que j'écris moi-même ».
C'est sans danger et réversible.

💡 Astuce : la méthode sans rien changer

Vous pouvez aussi lancer un script une seule fois sans modifier les réglages, en tapant :
powershell -ExecutionPolicy Bypass -File "C:\Users\FORMATION 2022_\Desktop\outil-diagnostic-pc-24-options.ps1"

01

Les couleurs du thème

Ranger les couleurs dans des variables pour un rendu obsidienne / doré cohérent.

🎯 Objectif

Donner à l'outil son identité visuelle dès le départ.
On définit cinq couleurs une bonne fois pour toutes, puis on les réutilise partout.
Résultat : un menu lisible, des titres dorés, des succès en vert et des alertes en rouge.

La notion clé : la variable

Une variable est une boîte étiquetée dans laquelle on range une valeur pour la réutiliser.
En PowerShell, le nom d'une variable commence toujours par le symbole $ (dollar).

  • $variable = valeur
  • -ForegroundColor
  • commentaire #

Par exemple, $CTitre = "Yellow" range le mot « Yellow » dans la boîte $CTitre.
Ensuite, écrire -ForegroundColor $CTitre revient à écrire -ForegroundColor Yellow.
L'avantage : si un jour on change la couleur des titres, on ne la modifie qu'à un seul endroit.

Tout ce qui suit un # est un commentaire : une note pour l'humain, que PowerShell ignore.

Le code à ajouter

Recopiez ce bloc tout en haut de votre fichier Diagnostic-TSSR.ps1 (désormais vide), puis enregistrez.

Diagnostic-TSSR.ps1 
# ------------------------------------------------------------
#  VARIABLES DE COULEURS (theme obsidienne / dore)
# ------------------------------------------------------------

# Couleur du texte courant
$CTexte   = "Gray"
# Couleur des titres / accents (rendu dore dans la console)
$CTitre   = "Yellow"
# Couleur de la bordure d'entete

# Couleur des messages de succes
$COk      = "Green"
# Couleur des messages d'alerte / echec

02

Les fonctions d'affichage

Créer la bannière, les en-têtes encadrés et la pause clavier — les briques réutilisables.

🎯 Objectif

Fabriquer trois petits outils que l'on rappellera des dizaines de fois :
Show-Banner (le grand titre de l'outil), Show-Entete (un titre encadré pour chaque section)
et Wait-Pause (« appuyez sur une touche »).

La notion clé : la fonction

Une fonction est un bloc de code auquel on donne un nom, pour le réutiliser sans le réécrire.
On la déclare une fois avec function Nom { ... }, puis on l'appelle autant de fois qu'on veut en écrivant simplement son nom.

  • function Nom { }
  • param([string]$x)
  • "=" * 50
  • ReadKey

Deux astuces présentes ici :

  • param([string]$Titre) permet à la fonction de recevoir une information (ici un titre) au moment où on l'appelle.
  • "=" * 50 répète le caractère = cinquante fois pour dessiner une ligne.
    PowerShell sait multiplier du texte !
  • [System.Console]::ReadKey($true) attend l'appui d'une touche sans l'afficher
    — parfait pour une pause propre.

Le code à ajouter

Ajoutez ces fonctions à la suite de vos variables de couleurs.

Diagnostic-TSSR.ps1 
# ------------------------------------------------------------
#  FONCTIONS UTILITAIRES
# ------------------------------------------------------------

# --- Affiche le titre de l'option choisie encadre ---
function Show-Entete {
    # Recupere le texte du titre passe en parametre
    param([string]$Titre)
    # Ligne vide avant l'entete pour aerer l'affichage
    Write-Host ""
    # Ligne superieure de l'encadre
    Write-Host (" " + ("=" * 50)) -ForegroundColor $CBordure
    # Affiche le titre en couleur d'accent
    Write-Host "  $Titre" -ForegroundColor $CTitre
    # Ligne inferieure de l'encadre
    Write-Host (" " + ("=" * 50)) -ForegroundColor $CBordure
}

# --- Affiche la banniere generale de l'outil ---
function Show-Banner {
    Write-Host (" " + ("=" * 50)) -ForegroundColor $CBordure
    Write-Host "  OUTIL DE DIAGNOSTIC SYSTEME - WINDOWS 10/11" -ForegroundColor $CTitre
    Write-Host "   Centre de formation MODE 83 - Draguignan" -ForegroundColor $CTexte
    Write-Host (" " + ("=" * 50)) -ForegroundColor $CBordure
}

# --- Pause avant de revenir au menu ---
function Wait-Pause {
    Write-Host ""
    Write-Host " Appuyez sur n'importe quelle touche pour selectionner une nouvelle option..." -ForegroundColor $CBordure
    # Attend l'appui d'une touche sans afficher de caractere
    $null = [System.Console]::ReadKey($true)
}

03

Première vraie information système

Interroger Windows pour afficher le nom de la machine, le système, la RAM, l'heure de démarrage.

🎯 Objectif

Écrire la première section de diagnostic réelle, Show-Infos.
Au lieu d'afficher du texte fixe, on va demander à Windows ses informations et les afficher.

Les notions clés

  • Get-CimInstance
  • $objet.Propriete
  • $env:COMPUTERNAME
  • [math]::Round
  • 1GB
  • $(...)

Get-CimInstance est la commande qui interroge Windows.
Par exemple Get-CimInstance Win32_OperatingSystem renvoie un objet décrivant le système.
On range cet objet dans une variable, puis on lit ses propriétés avec un point :
$os.Caption donne le nom de Windows,
$os.Version sa version.

Autres points utiles :

  • $env:COMPUTERNAME est une variable d'environnement Windows qui contient le nom du PC.
  • 1GB est un raccourci PowerShell valant 1 073 741 824 octets.
    Diviser des octets par 1GB donne des Go.
  • [math]::Round(valeur, 1) arrondit à 1 chiffre après la virgule.
  • Dans un texte, $(...) permet d'insérer le résultat d'un calcul : "Version : $($os.Version)".

Afficher la date et l'heure en français

Par défaut, la propriété $os.LastBootUpTime contient une date (un objet de type DateTime).
Si on l'affiche telle quelle, PowerShell la présente selon les réglages de la machine, ce qui peut donner un format anglais ou peu lisible.
Pour forcer un affichage 100 % français (jour et mois écrits en toutes lettres), on transforme cette date en texte avec la méthode .ToString(...).

La ligne utilisée est la suivante :

  • .ToString('dddd dd MMMM yyyy HH:mm:ss', ...) demande de fabriquer un texte selon un modèle que l'on choisit.
    Chaque lettre du modèle est un code remplacé par une vraie valeur :
  • dddd = le jour de la semaine en toutes lettres (par exemple lundi).
  • dd = le numéro du jour sur 2 chiffres (par exemple 06).
  • MMMM = le mois en toutes lettres (par exemple juillet).
    Attention : MM en majuscules désigne le mois ; mm en minuscules désignerait les minutes.
  • yyyy = l'année sur 4 chiffres (par exemple 2026).
  • HH = l'heure sur 24 heures (de 00 à 23).
    En majuscules, donc pas d'AM/PM.
  • mm = les minutes, ss = les secondes.

Le deuxième élément, [System.Globalization.CultureInfo]::GetCultureInfo('fr-FR'), est la partie qui dit : « écris ce texte selon les règles de la langue française ».
C'est lui qui garantit lundi et juillet plutôt que Moday et July, même si Windows est installé en anglais.
Le terme fr-FR signifie « français de France ».

Au final, l'instruction complète $($os.LastBootUpTime.ToString('dddd dd MMMM yyyy HH:mm:ss', [System.Globalization.CultureInfo]::GetCultureInfo('fr-FR'))) produit un résultat du type : lundi 06 juillet 2026 09:30:00.
Le $(...) autour sert, comme vu plus haut, à insérer ce texte directement dans la phrase affichée par Write-Host.

Le code à ajouter

Ajoutez cette fonction après vos fonctions utilitaires.
La ligne de test à la fin l'appelle directement pour vérifier le rendu.

Diagnostic-TSSR.ps1 
# ------------------------------------------------------------
#  FONCTIONS DE DIAGNOSTIC
# ------------------------------------------------------------

# --- 1. Informations systeme ---
function Show-Infos {
    Show-Entete "1. INFORMATIONS SYSTEME"
    # Recupere les informations du systeme d'exploitation
    $os = Get-CimInstance Win32_OperatingSystem
    # Recupere les informations materielles
    $pc = Get-CimInstance Win32_ComputerSystem
    Write-Host " Machine      : $env:COMPUTERNAME" -ForegroundColor $CTexte
    Write-Host " Systeme      : $($os.Caption)" -ForegroundColor $CTexte
    Write-Host " Version      : $($os.Version)" -ForegroundColor $CTexte
    # Calcule la memoire RAM totale en Go (arrondie a 1 decimale)
    $ram = [math]::Round($pc.TotalPhysicalMemory / 1GB, 1)
    Write-Host " Memoire (Go) : $ram" -ForegroundColor $CTexte
    Write-Host " Demarre le   : $($os.LastBootUpTime.ToString('dddd dd MMMM yyyy HH:mm:ss', [System.Globalization.CultureInfo]::GetCultureInfo('fr-FR')))" 
    Wait-Pause
}

04

Les tableaux : disque et processus

Découvrir le « pipeline » pour transformer des données en tableaux propres.

🎯 Objectif

Ajouter deux sections qui affichent des tableaux :
l'espace disque (Show-Disques) et
les 10 processus qui consomment le plus de mémoire (Show-Processus).
Ici, nous apprenons à utiliser l'outil le plus puissant de PowerShell : lepipeline.

La notion clé : le pipeline |

Le symbole | (barre verticale, « pipe ») envoie le résultat d'une commande vers la suivante,
comme un tapis roulant.
On lit chaque étape de gauche à droite, de haut en bas.

  • pipeline |
  • Where-Object
  • Sort-Object
  • Select-Object
  • Format-Table -AutoSize
  • @{ N=...; E={...} }

Les maillons les plus utiles :

  • Where-Object { condition } — ne garde que les éléments qui respectent la condition (un filtre).
  • Sort-Object Propriete -Descending — trie (ici du plus grand au plus petit).
  • Select-Object -First 10 — ne garde que les 10 premiers, et choisit les colonnes.
  • Format-Table -AutoSize — affiche un beau tableau aux colonnes ajustées.

La colonne calculée @{ N = "Total(Go)"; E = { [math]::Round($_.Size / 1GB, 1) } } mérite une explication :
N est le Nom de la colonne, E est l'Expression qui la calcule.
À l'intérieur, $_ (« dollar souligné ») désigne l'élément en cours de traitement dans le pipeline.
C'est la pièce qui déroute le plus au début : retenez simplement « $_ = la ligne actuelle ».

Le code à ajouter

Ajoutez ces deux fonctions à la suite de Show-Infos.

Diagnostic-TSSR.ps1 
# --- 2. Espace disque ---
function Show-Disques {
    Show-Entete "2. ESPACE DISQUE"
    # Recupere les disques locaux (DriveType 3 = disque fixe)
    Get-CimInstance Win32_LogicalDisk -Filter "DriveType=3" |
        Select-Object DeviceID,
            @{ N = "Total(Go)"; E = { [math]::Round($_.Size / 1GB, 1) } },
            @{ N = "Libre(Go)"; E = { [math]::Round($_.FreeSpace / 1GB, 1) } },
            @{ N = "% Libre";   E = { if ($_.Size) { [math]::Round($_.FreeSpace / $_.Size * 100, 0) } else { 0 } } } |
        Format-Table -AutoSize
    Wait-Pause
}

# --- 3. Top 10 des processus par memoire ---
function Show-Processus {
    Show-Entete "3. TOP 10 PROCESSUS (memoire)"
    Get-Process |
        Sort-Object WorkingSet -Descending |
        Select-Object -First 10 ProcessName,
            Id,
            @{ N = "Mem(Mo)"; E = { [math]::Round($_.WorkingSet / 1MB, 1) } } |
        Format-Table -AutoSize
    Wait-Pause
}

05

Le menu interactif

Afficher un menu, lire le choix de l'utilisateur et appeler la bonne fonction — en boucle.

🎯 Objectif

Transformer la collection de fonctions en un vrai outil navigable.
On affiche un menu, on lit le numéro tapé, on lance le diagnostic correspondant,
puis on revient au menu — jusqu'à ce que l'utilisateur tape Q pour quitter.
À ce stade, le menu ne propose que les 3 premières options : on l'agrandira en phase 6.

Les notions clés

  • Read-Host
  • switch ($x) { }
  • do { } while ( )
  • Clear-Host
  • $choix = Read-Host " Votre choix" — affiche une invite et range la saisie du clavier dans $choix.
  • switch ($choix) { "1" { ... } "2" { ... } }
    — un aiguillage : selon la valeur, on exécute le bloc correspondant.
    Bien plus lisible qu'une cascade de if.
  • do { ... } while ($choix -ne "Q") — répète le bloc tant que le choix n'est pas « Q ».
    -ne signifie « différent de » (not equal).
  • Clear-Host — efface l'écran à chaque tour, pour un affichage net.

Le code à ajouter

Ajoutez d'abord la fonction Show-Menu à la suite de vos diagnostics…

Diagnostic-TSSR.ps1 
# ------------------------------------------------------------
#  MENU PRINCIPAL
# ------------------------------------------------------------
function Show-Menu {
    Write-Host ""
    Write-Host "   [1]  Informations systeme" -ForegroundColor $CTexte
    Write-Host "   [2]  Espace disque" -ForegroundColor $CTexte
    Write-Host "   [3]  Top 10 processus (memoire)" -ForegroundColor $CTexte
    Write-Host ""
    Write-Host "   [Q]  Quitter" -ForegroundColor $CAlerte
    Write-Host ""
}

…puis ajoutez la boucle de navigation tout en bas du fichier (c'est elle qui démarre l'outil) :

Diagnostic-TSSR.ps1 
# ------------------------------------------------------------
#  BOUCLE DE NAVIGATION PRINCIPALE
# ------------------------------------------------------------
do {
    # Efface l'ecran pour un affichage propre a chaque tour
    Clear-Host
    Show-Banner
    Show-Menu
    # Lit le choix saisi par l'utilisateur
    $choix = Read-Host " Votre choix"
    # Aiguille vers la fonction correspondant au choix
    switch ($choix) {
        "1" { Show-Infos }
        "2" { Show-Disques }
        "3" { Show-Processus }
        "Q" { Write-Host ""; Write-Host " Fermeture de l'outil. Merci d'avoir utilise MODE 83" -ForegroundColor $CTitre }
        default {
            Write-Host ""
            Write-Host " Choix invalide, veuillez reessayer." -ForegroundColor $CAlerte
            Start-Sleep -Seconds 1
        }
    }
}
# Repete tant que l'utilisateur ne saisit pas Q
while ($choix -ne "Q")

06

Tous les autres diagnostics

Compléter l'outil avec les 21 sections restantes
— toujours la même structure, désormais familière.

🎯 Objectif

Ajouter les diagnostics 4 à 24 (services, réseau, ping, mises à jour, sécurité, BIOS, disques…).
Bonne nouvelle : toutes ces fonctions reposent sur ce que vous savez déjà
(en-tête → pipeline / objets → pause).
On introduit juste quatre nouvelles petites notions, puis on copie le reste.

Les nouvelles notions de cette phase

  • try { } catch { }
  • ForEach-Object
  • switch (valeur)
  • Read-Host (sous-choix)
try { ... } catch { ... } — lecture protégée
Certaines informations exigent des droits Administrateur ou n'existent pas sur tous les postes.
try tente l'opération , si elle échoue, catch affiche un message propre au lieu d'un gros message d'erreur rouge.
On l'associe à -ErrorAction Stop pour que la moindre erreur bascule dans le catch.
ForEach-Object { ... } — répéter pour chaque élément
Quand on a plusieurs éléments (plusieurs cartes réseau, plusieurs batteries),
ForEach-Object exécute le même bloc pour chacun.
À l'intérieur, $_ désigne l'élément courant, comme dans une colonne calculée.
switch ($valeur) { 2 { "..." } } — traduire un code
Windows renvoie souvent des codes numériques (état de batterie : 2 = sur secteur).
Un switch sert ici à traduire le code en texte lisible.

Read-Host en sous-menu
Certaines sections posent une petite question avant d'agir (services en cours / arrêtés, cible du ping…).
On réutilise Read-Host pour lire ce sous-choix.

Exemple commenté 1 — Services Windows (sous-choix + filtre)

Diagnostic-TSSR.ps1 
# --- 4. Services Windows ---

🎯 Objectif

Les notions clés

  • boucle imbriquee
  • .ToUpper()
  • [math]::Floor

On entoure toute la boucle de navigation par une seconde boucle do { ... } while ($refaire -eq "O").
La méthode .ToUpper() met la réponse en majuscule pour accepter aussi bien o que O.
La fonction Get-LigneCentree centre du texte dans l'encadré grâce à [math]::Floor (arrondi à l'entier inférieur).

# --- Centre un texte dans une largeur donnee ---
function Get-LigneCentree {
    param([string]$Texte, [int]$Largeur = 56)
    $espaces = $Largeur - $Texte.Length
    if ($espaces -lt 0) { $espaces = 0 }
    $gauche = [math]::Floor($espaces / 2)
    $droite = $espaces - $gauche
    return (" " * $gauche) + $Texte + (" " * $droite)
}

# --- Affiche le message de remerciement final ---
function Show-Remerciement {
    Clear-Host
    $L = 56
    Write-Host ""; Write-Host ""
    Write-Host ("    +" + ("-" * ($L + 2)) + "+") -ForegroundColor $CBordure
    Write-Host ("    | " + (Get-LigneCentree "" $L) + " |") -ForegroundColor $CBordure
    Write-Host ("    | " + (Get-LigneCentree "M E R C I !" $L) + " |") -ForegroundColor $CTitre
    Write-Host ("    | " + (Get-LigneCentree "" $L) + " |") -ForegroundColor $CBordure

Surprise pour beaucoup :
un fichier .xlsx est en réalité un fichier ZIP.
Si vous renommez un classeur Excel en .zip et que vous l’ouvrez, vous découvrez à l’intérieur plusieurs petits fichiers texte au format XML.
C’est la norme OpenXML.
Chaque fichier décrit une partie du classeur :

  • [Content_Types].xml → le sommaire
  • xl/workbook.xml → les feuilles
  • xl/styles.xml → couleurs & polices
  • xl/worksheets/sheet1.xml → les cellules

Donc, pour fabriquer un .xlsx nous-mêmes, il suffit de :
(1) écrire ces fichiers XML sous forme de texte, puis
(2) les ranger dans une archive ZIP que l’on nomme .xlsx.
Aucun besoin d’Excel installé, ni du module ImportExcel.
PowerShell 5.1 sait créer des ZIP grâce à la classe .NET System.IO.Compression.ZipArchive,
présente d’origine sous Windows 10/11.

Le piège des accents : l’UTF-8 sans BOM

Quand on écrit du texte français (é, è, à, ç…) dans les fichiers XML, il faut le bon encodage.
Excel attend de l’UTF-8 sans BOM (BOM = trois octets invisibles que certains outils ajoutent en tête de fichier).
Si on laisse le BOM, Excel affiche des caractères bizarres à la place des accents.
On force donc l’encodage avec New-Object System.Text.UTF8Encoding($false) — le $false signifie « sans BOM ».
C’est tout le secret des accents propres dans le fichier final.

Dans la phrase « Excel attend de l'UTF-8 sans BOM», BOM signifie Byte Order Mark (marque d'ordre des octets).
Il s'agit d'une courte séquence d'octets placée au tout début d'un fichier texte pour indiquer son encodage. Pour un fichier UTF-8, le BOM correspond aux trois octets suivants : EF BB BF

📌 RÈGLE D’OR — tout export va sur le Bureau, nulle part ailleurs

C’est une exigence absolue de l’outil : chaque fichier Excel doit être créé sur le Bureau de l’utilisateur, jamais ailleurs sur le disque.
Pour obtenir le chemin du Bureau de façon fiable — même si le Bureau est redirigé vers OneDrive, même si la session s’appelle autrement — on utilise :

$bureau = [Environment]::GetFolderPath("Desktop")

Cette commande Windows renvoie toujours le vrai chemin du Bureau courant.
On ne code donc jamais un chemin « en dur » comme C:\Users\…\Desktop.
Ensuite, on assemble le nom complet du fichier avec Join-Path.
Tous les exports de cet outil passent par cette règle.

  • ZipArchive

Étape 1 — Les 4 petits outils de base

On commence par quatre fonctions utilitaires courtes.

À quoi servent-elles ?
ConvertTo-XmlSafe remplace les caractères interdits en XML (&, <, >…) pour ne pas casser le fichier.
Get-ColonneExcel traduit un numéro de colonne (1, 2, 3…) en lettre Excel (A, B, C… AA).
Test-EstNumerique détecte si une valeur est un vrai nombre (pour l’aligner à droite).

# --- Echappe les caracteres speciaux interdits en XML ---
function ConvertTo-XmlSafe {
    # Recupere le texte a securiser
    param([string]$Texte)
    # Renvoie une chaine vide si l'entree est nulle
    if ($null -eq $Texte) { return "" }
    # Remplace l'esperluette en premier (sinon double encodage)
    $Texte = $Texte -replace '&', '&amp;'
    # Remplace le chevron ouvrant
    $Texte = $Texte -replace '<', '&lt;'
    # Remplace le chevron fermant
    $Texte = $Texte -replace '>', '&gt;'
    # Remplace le guillemet double
    $Texte = $Texte -replace '"', '&quot;'
    # Remplace l'apostrophe
    $Texte = $Texte -replace "'", '&apos;'
    # Supprime les caracteres de controle non valides en XML 1.0
    $Texte = [System.Text.RegularExpressions.Regex]::Replace($Texte, '[\x00-\x08\x0B\x0C\x0E-\x1F]', '')
    # Retourne le texte nettoye
    return $Texte
}

# --- Convertit un index de colonne (1..N) en lettre Excel (A, B, ... AA) ---
function Get-ColonneExcel {
    # Recupere l'index numerique de la colonne
    param([int]$Index)
    # Initialise la chaine resultat
    $lettre = ""
    # Boucle de conversion base 26 facon tableur
    while ($Index -gt 0) {
        # Calcule le reste pour la lettre courante
        $mod = ($Index - 1) % 26
        # Prefixe la lettre correspondante
        $lettre = [char](65 + $mod) + $lettre
        # Passe au rang superieur
        $Index = [int](($Index - $mod - 1) / 26)
    }
    # Retourne la reference de colonne
    return $lettre
}

# --- Determine si une valeur est un vrai nombre (pour alignement / type Excel) ---
function Test-EstNumerique {
    # Recupere la valeur a tester
    param($Valeur)
    # Verifie l'appartenance a un type numerique .NET (booleen exclu)
    return ($Valeur -is [byte]   -or $Valeur -is [sbyte]  -or `
            $Valeur -is [int16]  -or $Valeur -is [uint16] -or `
            $Valeur -is [int32]  -or $Valeur -is [uint32] -or `
            $Valeur -is [int64]  -or $Valeur -is [uint64] -or `
            $Valeur -is [single] -or $Valeur -is [double] -or $Valeur -is [decimal])
}

# --- Ajoute une entree texte (UTF-8 sans BOM) dans l'archive .xlsx ---
function Add-EntreeZip {
    # Recupere l'archive cible, le nom de l'entree et son contenu
    param($Archive, [string]$Nom, [string]$Contenu)
    # Cree l'entree compressee dans l'archive
    $entree = $Archive.CreateEntry($Nom, [System.IO.Compression.CompressionLevel]::Optimal)
    # Ouvre le flux d'ecriture de l'entree
    $flux = $entree.Open()
    # Prepare un encodeur UTF-8 SANS BOM (accents corrects pour Excel)
    $utf8 = New-Object System.Text.UTF8Encoding($false)
    # Convertit le contenu en octets UTF-8
    $octets = $utf8.GetBytes($Contenu)
    # Ecrit les octets dans l'entree
    $flux.Write($octets, 0, $octets.Length)
    # Ferme le flux de l'entree
    $flux.Close()
}

Étape 2 — Le moteur : Export-ExcelPremium

Voici le cœur du module.
Il prend un titre, un tableau d’objets (les données) et un chemin de fichier, puis fabrique le .xlsx complet : calcul des largeurs de colonnes, ligne de titre fusionnée, sous-titre (poste + date + MODE 83), en-têtes figés, zébrage des lignes, filtre automatique, thème obsidienne / or, puis écriture du ZIP.

Ce bloc est long et très technique (il contient le XML des styles Excel).
Recopiez-le sans le modifier : il fonctionne dès le premier essai.
L’important pour vous est d’avoir compris l’idée (un ZIP de fichiers XML, en UTF-8 sans BOM).

# --- Construit et ecrit un fichier .xlsx premium a partir d'objets ---
function Export-ExcelPremium {
    # Titre de la section, donnees a exporter et chemin du fichier
    param(
        [string]$Titre,
        [object[]]$Donnees,
        [string]$CheminFichier

    # Charge les assemblys de compression natifs (.NET 4.5+, present sur Win10/11)
    Add-Type -AssemblyName System.IO.Compression -ErrorAction SilentlyContinue
    Add-Type -AssemblyName System.IO.Compression.FileSystem -ErrorAction SilentlyContinue

    # Force le tableau (cas d'un objet unique renvoye par le collecteur)
    $Donnees = @($Donnees)
    # Determine les colonnes a partir du premier objet
    if ($Donnees.Count -gt 0 -and $Donnees[0]) {
        # Recupere les noms de proprietes dans l'ordre
        $colonnes = @($Donnees[0].PSObject.Properties.Name)
    } else {
        # Jeu de secours si aucune donnee disponible
        $colonnes = @("Information")
        $Donnees  = @([pscustomobject]@{ "Information" = "Aucune donnee disponible pour cette section." })
    }
    # Nombre de colonnes
    $nbCol = $colonnes.Count
    # Lettre de la derniere colonne
    $derniereColLettre = Get-ColonneExcel $nbCol

    # ---- Calcule des largeurs de colonnes (selon le contenu) ----
    # Construit le bloc <cols>
    $colsXml = '<cols>'
    # Parcourt chaque colonne
    for ($i = 0; $i -lt $nbCol; $i++) {
        # Longueur de l'en-tete comme base
        $maxLen = $colonnes[$i].Length
        # Compare avec la longueur de chaque valeur de la colonne
        foreach ($d in $Donnees) {
            $v = $d.$($colonnes[$i])
            if ($null -ne $v) {
                $l = ($v.ToString()).Length
                if ($l -gt $maxLen) { $maxLen = $l }
            }
        }
        # Largeur calculee, bornee entre 12 et 60
        $w = [math]::Min([math]::Max($maxLen * 1.15 + 3, 12), 60)
        # Formate la largeur en culture invariante (point decimal)
        $wStr = [string]::Format([System.Globalization.CultureInfo]::InvariantCulture, '{0:0.00}', $w)
        # Index 1-base de la colonne
        $idx = $i + 1
        # Ajoute la definition de colonne
        $colsXml += '<col min="' + $idx + '" max="' + $idx + '" width="' + $wStr + '" customWidth="1"/>'
    }
    # Ferme le bloc <cols>
    $colsXml += '</cols>'

    # ---- Construit les lignes de la feuille ----
    # Accumulateur performant pour le XML des lignes
    $sb = New-Object System.Text.StringBuilder

    # Ligne 1 : titre fusionne (style 1)
    [void]$sb.Append('<row r="1" ht="34" customHeight="1">')
    [void]$sb.Append('<c r="A1" s="1" t="inlineStr"><is><t xml:space="preserve">' + (ConvertTo-XmlSafe $Titre) + '</t></is></c>')
    for ($i = 2; $i -le $nbCol; $i++) {

    }
    [void]$sb.Append('</row>')

    # Ligne 2 : sous-titre fusionne (poste + date + organisme)

    [void]$sb.Append('<row r="2" ht="20" customHeight="1">')
    [void]$sb.Append('<c r="A2" s="2" t="inlineStr"><is><t xml:space="preserve">' + (ConvertTo-XmlSafe $sousTitre) + '</t></is></c>')
    for ($i = 2; $i -le $nbCol; $i++) {
        [void]$sb.Append('<c r="' + (Get-ColonneExcel $i) + '2" s="2"/>')
    }
    [void]$sb.Append('</row>')

    # Ligne 3 : en-tetes de colonnes (style 3)
    [void]$sb.Append('<row r="3" ht="24" customHeight="1">')
    for ($i = 0; $i -lt $nbCol; $i++) {
        [void]$sb.Append('<c r="' + (Get-ColonneExcel ($i + 1)) + '3" s="3" t="inlineStr"><is><t xml:space="preserve">' + (ConvertTo-XmlSafe $colonnes[$i]) + '</t></is></c>')
    }
    [void]$sb.Append('</row>')

    # Lignes de donnees (zebrage alterne)
    $rowNum = 4
    $zebra  = $false
    foreach ($d in $Donnees) {
        [void]$sb.Append('<row r="' + $rowNum + '">')
        for ($i = 0; $i -lt $nbCol; $i++) {
            # Reference de la cellule (ex: B5)
            $ref = (Get-ColonneExcel ($i + 1)) + $rowNum
            # Valeur de la cellule
            $val = $d.$($colonnes[$i])
            if (Test-EstNumerique $val) {
                # Cellule numerique : style aligne a droite (6 ou 7 si zebre)
                $sNum = if ($zebra) { '7' } else { '6' }
                # Formatage invariant (point decimal) pour Excel
                $numStr = [string]::Format([System.Globalization.CultureInfo]::InvariantCulture, '{0}', $val)
                [void]$sb.Append('<c r="' + $ref + '" s="' + $sNum + '"><v>' + $numStr + '</v></c>')
            } else {
                # Cellule texte : style aligne a gauche (4 ou 5 si zebre)
                $sTxt = if ($zebra) { '5' } else { '4' }
                # Texte securise pour XML (accents preserves)
                $txt = if ($null -eq $val) { '' } else { ConvertTo-XmlSafe ($val.ToString()) }
                [void]$sb.Append('<c r="' + $ref + '" s="' + $sTxt + '" t="inlineStr"><is><t xml:space="preserve">' + $txt + '</t></is></c>')
            }
        }
        [void]$sb.Append('</row>')
        # Ligne suivante et bascule du zebrage
        $rowNum++
        $zebra = -not $zebra
    }
    # Derniere ligne ecrite
    $derniereLigne = $rowNum - 1
    # XML complet des lignes
    $sheetData = $sb.ToString()

    # Filtre automatique sur la plage en-tetes + donnees
    $autoFilter = '<autoFilter ref="A3:' + $derniereColLettre + $derniereLigne + '"/>'
    # Fusion des deux bandeaux (titre + sous-titre)
    $merges = '<mergeCells count="2"><mergeCell ref="A1:' + $derniereColLettre + '1"/><mergeCell ref="A2:' + $derniereColLettre + '2"/></mergeCells>'

    # ---- Assemble la feuille de calcul (sheet1.xml) ----
    $sheetXml = '<?xml version="1.0" encoding="UTF-8" standalone="yes"?>' +
        '<worksheet xmlns="http://schemas.openxmlformats.org/spreadsheetml/2006/main" xmlns:r="http://schemas.openxmlformats.org/officeDocument/2006/relationships">' +
        '<dimension ref="A1:' + $derniereColLettre + $derniereLigne + '"/>' +
        '<sheetViews><sheetView showGridLines="0" tabSelected="1" workbookViewId="0">' +
        '<pane ySplit="3" topLeftCell="A4" activePane="bottomLeft" state="frozen"/>' +
        '<selection pane="bottomLeft" activeCell="A4" sqref="A4"/>' +
        '</sheetView></sheetViews>' +
        '<sheetFormatPr defaultRowHeight="15"/>' +
        $colsXml +
        '<sheetData>' + $sheetData + '</sheetData>' +
        $autoFilter +
        $merges +
        '</worksheet>'

    # ---- Parties statiques du classeur ----
    # Table des types de contenu
    $contentTypes = @'
<?xml version="1.0" encoding="UTF-8" standalone="yes"?><Types xmlns="http://schemas.openxmlformats.org/package/2006/content-types"><Default Extension="rels" ContentType="application/vnd.openxmlformats-package.relationships+xml"/><Default Extension="xml" ContentType="application/xml"/><Override PartName="/xl/workbook.xml" ContentType="application/vnd.openxmlformats-officedocument.spreadsheetml.sheet.main+xml"/><Override PartName="/xl/worksheets/sheet1.xml" ContentType="application/vnd.openxmlformats-officedocument.spreadsheetml.worksheet+xml"/><Override PartName="/xl/styles.xml" ContentType="application/vnd.openxmlformats-officedocument.spreadsheetml.styles+xml"/></Types>
'@
    # Relations racine
    $relsRoot = @'
<?xml version="1.0" encoding="UTF-8" standalone="yes"?><Relationships xmlns="http://schemas.openxmlformats.org/package/2006/relationships"><Relationship Id="rId1" Type="http://schemas.openxmlformats.org/officeDocument/2006/relationships/officeDocument" Target="xl/workbook.xml"/></Relationships>
'@
    # Definition du classeur
    $workbook = @'
<?xml version="1.0" encoding="UTF-8" standalone="yes"?><workbook xmlns="http://schemas.openxmlformats.org/spreadsheetml/2006/main" xmlns:r="http://schemas.openxmlformats.org/officeDocument/2006/relationships"><sheets><sheet name="Diagnostic" sheetId="1" r:id="rId1"/></sheets></workbook>

# --- 1. Informations systeme (cle / valeur) ---
function Get-DataInfos {
    $os = Get-CimInstance Win32_OperatingSystem
    $pc = Get-CimInstance Win32_ComputerSystem
    $ram = [math]::Round($pc.TotalPhysicalMemory / 1GB, 1)
    @(
        [pscustomobject]@{ "Propriete" = "Machine";      "Valeur" = "$env:COMPUTERNAME" }
        [pscustomobject]@{ "Propriete" = "Systeme";      "Valeur" = "$($os.Caption)" }
        [pscustomobject]@{ "Propriete" = "Version";      "Valeur" = "$($os.Version)" }
        [pscustomobject]@{ "Propriete" = "Memoire (Go)"; "Valeur" = "$ram" }
        [pscustomobject]@{ "Propriete" = "Demarre le";   "Valeur" = "$($os.LastBootUpTime)" }

# --- 2. Espace disque ---
function Get-DataDisques {
    Get-CimInstance Win32_LogicalDisk -Filter "DriveType=3" |
        Select-Object @{ N = "Lecteur";    E = { $_.DeviceID } },
            @{ N = "Total (Go)"; E = { [math]::Round($_.Size / 1GB, 1) } },
            @{ N = "Libre (Go)"; E = { [math]::Round($_.FreeSpace / 1GB, 1) } },
            @{ N = "% Libre";    E = { if ($_.Size) { [math]::Round($_.FreeSpace / $_.Size * 100, 0) } else { 0 } } }
}

# --- Construit un nom de fichier ASCII propre a partir du titre ---
function Get-NomFichierSain {
    # Recupere le libelle de la section
    param([string]$Texte)
    # Decompose les caracteres accentues (forme NFD)
    $decompose = $Texte.Normalize([System.Text.NormalizationForm]::FormD)
    # Retire les marques diacritiques (accents)
    $sansAccents = [System.Text.RegularExpressions.Regex]::Replace($decompose, '\p{Mn}', '')
    # Remplace tout caractere non alphanumerique par un tiret
    $propre = [System.Text.RegularExpressions.Regex]::Replace($sansAccents, '[^A-Za-z0-9]+', '-')
    # Supprime les tirets en debut/fin
    return $propre.Trim('-')
}

Bonus : compiler l'outil en .exe

Transformer le script en un programme autonome que l'on lance d'un double-clic, sans ouvrir PowerShell.

🎯 Objectif

Produire un fichier Diagnostic-TSSR.exe distribuable :
vous pourrez lancer l'outil d'un simple double-clic, même sur un poste où l'exécution de scripts est bloquée.
Cette étape est optionnelle : le script .ps1 fonctionne déjà parfaitement.

La notion clé : ps2exe

ps2exe est un module gratuit qui emballe un script PowerShell dans un véritable exécutable Windows.
Le code reste le même : ps2exe ajoute juste une « coquille » .exe autour.
On l'installe une seule fois depuis la PowerShell Gallery (le dépôt officiel de Microsoft).

  • Install-Module
  • Invoke-ps2exe
  • -inputFile / -outputFile

Étape 1 — Installer ps2exe (une seule fois)

Ouvrez PowerShell et tapez la commande ci-dessous.
À la première utilisation de la Gallery, Windows demande d'autoriser le dépôt :
répondez O (Oui)
ou A (Oui pour tout).

Console PowerShell 
# Installe le module ps2exe pour l'utilisateur courant (sans droits admin)
Install-Module -Name ps2exe -Scope CurrentUser -Force
💡 Si l'installation est refusée

Sur certains postes, il faut d'abord autoriser les téléchargements sécurisés.
Lancez une seule fois : [Net.ServicePointManager]::SecurityProtocol = [Net.SecurityProtocolType]::Tls12 puis relancez l'installation.
En formation, c'est rarement nécessaire sous Windows 10/11 à jour.

Placez-vous dans le dossier qui contient Diagnostic-TSSR.ps1 (avec cd), puis lancez la compilation.
On en profite pour renseigner le titre, l'éditeur et le produit : ces informations apparaîtront dans les propriétés du fichier .exe.
